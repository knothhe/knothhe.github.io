港卡

港卡是我很久之前就想办的，不过一直拖着，凑巧和一个朋友聊了之后，他也有办港卡的想法，于是就约定九月底，国庆之前去趟香港，办一下需要的港卡。网络上办理港卡的教程还是比较多的，整体上还是比较顺利的，当然也踩了一些坑。我这里列出来一些比较重要的点。

我主要办理了四张港卡，两张传统银行，中银香港和汇丰，两家数字银行，众安和蚂蚁。这四家银行都支持线上办理，所以我们也就都通过线上方式开通，没有去线下的银行办理。

  1. 办理前需要下载好对应银行的 APP，在手机上操作申请。中银香港是BOCHK中银香港，汇丰是 HSBC HK，众安是 ZA Bank，蚂蚁是 ANT BANK，不过蚂蚁其实也可以不办理。
  2. 银行应用都会检测代理和 Root，确保安全的 APP 环境，所以最好是人在香港，使用没有越狱过的 iPhone 或者没有 Root 过的安卓申请。
  3. 值得一提的是汇丰需要手机支持 NFC，申请过程中会通过 NFC 读取证件信息，我使用的 iPhone 是没有问题的。
  4. 非常重要的事情，如果汇丰在注册成功后，出现注册手机银行的步骤，千万要注册，否则就要像我一样，需要等到密码函才能注册手机银行。
  5. 中银香港的手机银行非常坑爹的帐号密码设定，注册中银手机银行的时候，密码是有限定要求的，但是注册过程中不会提示报错，等注册完成后，如果不符合中银的密码规范，用注册时的密码会登录不上手机银行，这个时候需要根据中银密码的要求，把注册时的密码修改成符合中银要求的密码才能登录成功。我生成的密码因为包含了 { 这个字符，登录的时候需要去掉 { 才能成功登录。第一天登录的时候我就很纳闷为什么登录不成功，真没想到是踩了这个坑。

密码须包含数字、英文（大写）、英文（小写）及符号中至少两种类型的字符，但不能包括三个或以上连续相同的字元，有效字数为8至12位，大写小写属于不同字符。 密码可包含下列字符： i) 数字 ii) 英文（大写） iii) 英文（小写） iv)符号 # $ % & ( ) * + , - . : ; ? @ = _ < >

  1. 线上申请需要人在香港，主要是需要出入境记录，和香港 wifi。出入境记录可以到移民局小程序查询，到香港之后很快就有记录，我们大概一个小时就查询到了记录，需要注意的是，只会有一条大陆的出境记录，不会有香港的入境记录。离港后，会有一条大陆的入境记录。

我朋友因为是两字的姓名，汇丰当场没有通过，离港回来之后很快也就邮件通知通过了。我因为没有当场注册手机银行的原因，和他一样，要等密码函。

我们是 9.20 左右办理的港卡，中银香港和汇丰的银行卡，在办理之后的半个月之内我就收到了，但是汇丰的密码函到今天还没有收到。

Omarchy

Omarchy 是 DHH 创办的一个基于 Arch-linux 的新的发行版，我是因为不想每天背着 macbook 上下班，就想着在家里的电脑上装一个 linux 系统平时使用，和 Windows 系统组成双系统。Windows 系统平时用来玩游戏，linux 则用来办公之类的。目前用下来的体验是，linux 系统还是不如 Windows 和 macOS 稳定，总是会出现各种各样的问题。我碰到的问题不限于下面这些。

  1. 输入法问题。Linux 系统下仍然没有足够好用的中文输入法，我现在使用的是 fctix5 + rime，和 macOS 上使用的 suqirrel 都属于 rime 输入法在不同操作系统上的实现。我发现中文输入法虽然能用，但是会出现一些渲染问题，虽然不影响使用。一开始我还以为是应用程序的问题，后来才发现是输入法的问题。最难以忍受的是，linux、vim 插件、中文输入法，这三者一起使用的时候，时不时出现一些输入问题，非常打断输入体验，所以这篇问题是使用 neovim 编写的。
  2. 应用崩溃。经常性不知道因为什么操作，应用就会崩溃，无法再次打开，一般这种时候我只能重且解决。
  3. 网络问题。不知道什么原因，在 linux 上访问 B 站会非常的卡顿，这里可以排除网络原因。同样的网络和硬件，我在 Windows 系统上没有出现这个问题，而且在同一网络下的 macOS 也没有出现这个问题。

以上的问题，我都不知道原因，目前也没有解决，在将就用着。

不过嘛，既然是我选择了 Omarchy，我也就得接受这些缺点，不然为什么不用 Windows 或者 macOS 呢。而且有点也是有的，命令行使用很舒服，hyprland 平铺窗口管理器，基于文件的配置等等。

chezmoi

由于多了一台系统，我也就考虑起配置文件同步和备份的问题了。先前我是自己编写脚本，通过软链接的方式管理配置文件的，不能说没法使用，只能说是有点糟糕，所以这次就尝试通过 chezmoi 解决这个问题了。我这里不会详细说明 chezmoi 的使用方式，不过还是很推荐大家去尝试使用一下的。一些特别的点列在这里。

  1. 所有 $HOME 目录下的配置文件都可以使用 chezmoi 管理。
  2. 可以嵌入密钥，比如我使用 pass 管理密码和密钥，就可通过 chezmoi 的模板 tmpl 嵌入指令 {{ pass secret/name }}，在手动 chezmoi apply 应用之后，就会解密成对应的值。
  3. 可以加密管理文件，避免敏感文件泄露，我配置的 gpg 方式加解密文件，本身 pass 也是 gpg 的脚本封装，使用 gpg 也就理所当然了。
  4. git 管理，这当然是基础之中的基础。
  5. 我可以在 chezmoi 这个仓库里，使用 claude code 帮我编写各种需要的脚本。
  6. 同一个文件可以管理多平台配置。当同一个文件需要管理多平台配置是，可以使用 chezmoi 的 tmpl 文件，针对不同平台使用指令编写不同的配置内容。

缺点也是有的，还是需要学习，不过我通过官网，大模型和开发者的 chezmoi 配置仓库，已经能够搭建出适合我自己使用的配置文件管理方式了，强烈推荐。

空洞骑士

国庆假期，我都在家玩游戏，一般时间在玩空洞骑士，另一半时间在玩空洞骑士丝之歌，简单总结就是在受苦，不知道红温了多少次。空洞骑士我是几年前就有尝试过一次，当时玩到三螳螂那里，就发现自己实在是手残，没办法通过三螳螂，也就弃坑了空洞骑士。今年的 9.4，续作丝之歌发售之后，我就想着，还是试一试空洞骑士，根据攻略竟然也通关了前作，虽然还是很手残。虽然这次玩也还是手残，60H 的游戏时间，大概有 20H 在 竞技场3 上，虽然打完我才之后，这不是主线必须要过的，😭。

不过我还是高估了自己，后半段时间玩丝之歌，时不时红温，现在就是尽量让自己心态好一点，慢慢玩吧，能玩一点是一点，实在太手残，玩不下去就算了。

另外的一点是，丝之歌这作，我感觉制作组的恶意实在是太大了。

  1. 打完 boss/车轮战 之后奖励三小时的跳跳乐。
  2. 跳跳乐的过程中，一定要放恶心人的小怪。
  3. 打 boss 之前，居然还要经过一段有小怪的跳跳乐。
  4. 走着走着，就被不知名小怪打死了。

我玩游戏的时候大部分都是负反馈，看主播玩丝之歌，就不少正反馈了。

不过丝之歌的美术，音乐设计的我还是挺喜欢的。
